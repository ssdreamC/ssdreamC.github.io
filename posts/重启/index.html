重启

先进入堡垒机

堡垒机的地址是https://172.16.11.12/#/desktop，然后需要重启的服务器是华为超融合_cms1，172.16.16.67

  1. 先是如下命令杀死进程：
# 切换到root用户
su root # 打完这条命令，回车后输入root的密码，屏幕不会显示，输入完密码直接按回车就行
# 但密码我还不知道啊，谁给个密码啊

# 杀死java进程
killall java

查看是否还有 java 进程，反正看不看都要重启了要不就不看了？？？

# 下面是查看是否有java进程，按照新胜的说法，应该是还会有别的杀不死的僵尸进程
ps aux | grep java

  1. 然后重启服务器：

    reboot

重启后配置

  1. 挂载

重新启动后，别忘了先切换到root用户

# 切换到root用户
su root

这些命令将 GlusterFS 节点中的三个卷（repo_gv0、webapp_gv1、webapp_gv2）挂载到本地服务器的三个不同目录中

mount -t glusterfs CMS-SERVER-01:/repo_gv0 /data/UCMSServer/repo 
mount -t glusterfs CMS-SERVER-01:/webapps_gv1 /data/UCMSServer/tomcat/webapps 
mount -t glusterfs CMS-SERVER-01:/templates_gv2 /data/UCMSServer/templates
  1. 系统启动后需要删除一些文件

一共两个目录下的内容需要删除

# 先进入第一个目录
cd /data/UCMSServer/appserver/apache-activemq-5.16.1/data
# 删除该目录下所有内容
rm -rf *

# 再进入第二个目录
cd /data/UCMSServer/cache
# 删除该目录下所有内容
rm -rf *

启动

  1. 启动 cms 相关
# 进入启动sms目录
cd /usr/UCMSServer/bin/03-startCms 
# 启 动 redis缓 存 
./01-redisStart.sh 
# 启 动 JMS队 列 
./02-jmsStart.sh 
# 启 动 CMS 
./03-cmsStart.sh 
# 启 动 发 布 
./04-pubStart.sh
  1. 启动 搜索 相关
# 进目录、执行命令，进目录、执行命令
cd /usr/UCMSSearch-es
./01-startEs.sh 
cd /usr/UCMSSearch-es
./02-start-xxl-job.sh 
# 进入es的目录
cd /usr/UCMSSearch-es/elasticsearch-head
# 以 elasticsearch 的身份执行命令
su elasticsearch -c "nohup ./grunt server &"
# 进目录，执行命令
cd /usr/UCMSServer/UCMSSearch-es/search-es
./03-start-searches-new.sh
  1. 启动 同步栏目树 相关
cd /data/UCMSServer/wcm 

# 下面的命令，一直复制到 &，别忘了把 & 也复制上
nohup /usr/UCMSServer/common/j2sdk_linux/bin/java -jar wcm-0.0.1-SNAPSHOT.jar & 
# 同上
cd /data/pkg/node_exporter && nohup ./node_exporter &
  1. 启动 防篡改 相关
/data/iguard/ilocker/bin/ilocker -f start 
/data/iguard/stagingd/admtool start
